# 디자인 시스템 — 퀀트플랫폼

웹앱과 로컬앱의 UI는 이 문서를 기준으로 한다. 새 화면·컴포넌트를 만들 때
색상·간격·패턴을 여기서 가져다 쓰고, 벗어날 때는 먼저 합의한다.

## 제품 맥락

- **무엇:** 한국 주식 자동매매 플랫폼. 웹앱에서 매매 전략을 수립하고,
  사용자 PC의 로컬앱이 그 전략으로 모의투자를 자동 실행한다.
- **대상:** 퀀트 트레이딩을 시작하려는 초중급 개인투자자.
- **표면:** 웹앱(React SPA, 전략 수립) + 로컬앱(Python Tkinter, 자동 실행).

## 디자인 원칙

1. **흐름을 보이게.** 전략 만들기 → 모의투자 배정 → 기기 연결 → 자동매매 가동.
   사용자는 늘 "지금 어디에 있고 다음에 뭘 할지" 알 수 있어야 한다.
2. **숫자를 해석해 준다.** 지표를 그대로 던지지 않는다. 등급·평이한 문장·
   툴팁으로 초중급 사용자가 판단할 수 있게 한다. (제품의 핵심 차별점)
3. **신뢰감.** 자동매매에 돈을 맡기는 도구다. 회색 기본 위젯이 아니라
   일관된 톤, 명확한 상태 표시, 빈 상태 안내로 신뢰를 준다.

## 아이덴티티

- **방향:** 클린 SaaS. 인디고 액센트 + 라이트 그레이 배경 + 카드형 패널.
- **장식 수준:** 미니멀. 타이포·여백·테두리가 일한다. 그라데이션·장식 없음.

## 색상

웹앱 토큰은 `web/src/index.css`의 `:root`, 로컬앱 상수는
`local/localapp/gui.py` 상단에 동일 값으로 정의되어 있다.

| 역할 | HEX | 용도 |
|------|-----|------|
| bg | `#f4f5f7` | 페이지 배경 |
| panel | `#ffffff` | 패널·카드 |
| border | `#e3e5ea` | 테두리·구분선 |
| text | `#1a1d23` | 본문 |
| muted | `#6b7280` | 보조 텍스트 |
| accent | `#4f46e5` | 인디고 — 주요 액션, 활성 상태 |
| accent-soft | `#eef2ff` | 액센트 배경 틴트 |
| green | `#15803d` | 수익·정상·완료 |
| red | `#b91c1c` | 손실·오류 |
| amber | `#b45309` | 주의·미완료 |
| green/red/amber-soft | `#ecfdf3` / `#fef2f2` / `#fffbeb` | 상태 카드 배경 틴트 |

의미 규칙: 수익/양호 = green, 손실/오류 = red, 주의/설정 미완료 = amber,
주요 액션·진행 중 단계 = accent.

## 타이포그래피

- **스택:** `-apple-system, "Segoe UI", "Malgun Gothic", Roboto, sans-serif`.
  한글(맑은 고딕)·영문(Segoe UI) 모두 OS 기본 폰트로 안정적. 별도 웹폰트 없음.
- **데이터/로그:** 등폭(`Consolas`) — 로컬앱 로그 영역.
- **크기:** 페이지 제목 22 / 패널 제목 15 / 본문 13~14 / 보조 12 / 캡션 11.

## 간격·레이아웃

- 패널 `padding 20px`, 패널 간 `margin-bottom 18px`.
- 모서리 반경 `--radius: 10px` (작은 칩·버튼은 7~8px).
- 웹앱 본문 최대폭 1100px, 좌측 220px 사이드바.
- 로컬앱 창 560×740, 섹션 간 `padx 12 / pady 4~6`.

## 컴포넌트 패턴

- **패널(`.panel`):** 흰 배경 + 1px 테두리 카드. 모든 콘텐츠 묶음의 기본 단위.
- **지표 카드(`.stat`):** 라벨 + 큰 값. 값에 `.pos`/`.neg`로 손익 색.
- **지표 툴팁(`.metric-hint`):** 라벨 옆 `?` 호버 시 한 줄 설명. 전문용어
  (MDD·샤프·CAGR·p-value 등)에는 반드시 붙인다.
- **결과 해석 카드(`Verdict`):** 백테스트 결과 상단. 등급 뱃지 + 한 줄 요약 +
  항목별 코멘트(pt-good/pt-warn/pt-bad). tone(good/ok/warn/bad)로 좌측
  보더·배경 틴트 결정.
- **진행 단계(`.steps`/`.step`):** done(✓·green) / current(번호·accent 강조,
  CTA 노출) / todo(회색). 온보딩 여정을 항상 보이게 한다.
- **하단 액션바(`.action-bar`):** `position: sticky; bottom`. 주요 액션을
  스크롤과 무관하게 늘 닿게 한다. 기본 액션 1개 + 보조 액션.
- **빈 상태(`.empty-state`):** 회색 한 줄 금지. 제목 + 안내 문단 + CTA 버튼 카드.
- **버튼:** 기본 = accent 채움. 보조 = `.ghost`(연회색). 작은 건 `.sm`.
- **로컬앱:** ttk `clam` 테마 + 위 팔레트. 상단 상태 히어로(앰버=미완료 /
  그린=실행 중 / 슬레이트=중지), 섹션은 ①②③ 번호 + 헤더에 ✓ 진행 표시.

## 모션

미니멀-기능적. 색·투명도 전환만(120ms 내외). 등장 애니메이션·스크롤 연출 없음.

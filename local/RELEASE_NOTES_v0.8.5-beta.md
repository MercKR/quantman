## 주요 변경 — KIS 주문 API tr_id migrate (긴급)

v0.8.4 이전 사용한 국내주식 주문 tr_id들이 KIS 공식 spec ([국내주식]
주문_계좌.xlsx)에 명시되어 있지 않음. KIS가 grace 기간 동안 옛 tr_id를
받아주고 있어 동작은 했지만, KIS가 옛 형식 제거 시 주문 즉시 실패 위험.

## 변경된 tr_id 4개

| 기능 | v0.8.4 이전 | v0.8.5 (KIS 공식) |
|---|---|---|
| 국내 매수 | `TTTC0802U` / `VTTC0802U` | **`TTTC0012U` / `VTTC0012U`** |
| 국내 매도 | `TTTC0801U` / `VTTC0801U` | **`TTTC0011U` / `VTTC0011U`** |
| 국내 정정취소 | `TTTC0803U` / `VTTC0803U` | **`TTTC0013U` / `VTTC0013U`** |
| 국내 일별주문체결 | `TTTC8001R` / `VTTC8001R` | **`TTTC0081R` / `VTTC0081R`** (3개월 이내) |

endpoint, payload 구조 동일. tr_id만 변경.

## 사용자 액션 — 매우 권장
1. zip 덮어쓰기 후 로컬앱 재시작.
2. **모의투자 모드에서 "지금 한 번 실행" 1회 클릭**해서 정상 동작 확인.
3. cycle 로그·주문 로그에 `rt_cd=0` 정상 응답 확인.
4. 정상 확인 후 실전 매매에 사용.

## 변경 없음 (이미 KIS 공식 spec과 일치)
- 잔고 조회: `TTTC8434R` / `VTTC8434R` ✓
- 현재가 (REST): `FHKST01010100` ✓
- 실시간 체결가 (WebSocket): `H0STCNT0` ✓ (KRX 전용)
- 체결통보: `H0STCNI0` / `H0STCNI9` ✓

## 미반영 사항 (장기 검토)
- **NXT(넥스트레이드) 통합 시세**: 2025년 출범 한국 ATS. 실전 사용자만 사용
  가능한 `H0UNCNT0`(통합) 또는 `H0NXCNT0`(NXT 전용)는 모의투자 미지원이라
  현 베타 단계에선 미반영. 실전 사용자 50명+ 시점에 도입 검토.

## Server 호환
모든 server 버전 호환. 변경은 로컬앱 KIS REST API 호출만.
